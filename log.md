# 变更日志

## 2026-04-17 (街景沉浸模式与全屏 HUD)

### 主界面街景集成（从 demo 回灌到生产页面）
- 将街景能力从独立 `streetviewtest` 实验页迁移到主骑行界面 `rideDashboard`。
- 新增“加载街景 / 实时街景”入口：在开始骑行界面直接输入 API Key 并开启街景，不再依赖独立 demo 页面。
- 街景容器采用双层 `svPano1/svPano2`，与主地图共用同一舞台容器，支持平面地图与街景视图切换。

### 街景加载核心算法（抗黑屏）
- **时间节流**：街景更新周期固定为 `3s`，避免每秒切 pano 导致频繁重绘与闪烁。
- **轨迹插值**：根据当前累计距离在 GPX 点列中插值出实时经纬度与坡度。
- **朝向计算**：使用前方 `5m` 的 lookahead 点计算 heading，保证镜头朝向与骑行方向一致。
- **俯仰计算**：由坡度换算相机 pitch，公式为 `pitch = atan(grade / 100)`（角度制）。
- **最近 pano 查找**：通过 `StreetViewService.getPanorama({ location, radius: 50 })` 获取最近合法全景点。
- **双缓冲切换**：若目标 pano 与当前 pano 不同，先在隐藏层 `setPano` 预载；监听 `status_changed=OK` 后再进行透明度切换，避免黑屏。
- **前瞻预热**：按当前速度预测 `5s` 后位置并提前请求下一 pano，降低下一次切换延迟。

### 沉浸式 UI/HUD
- 实现街景全屏模式：隐藏 Dashboard 顶部标题区，街景主画面铺满视口。
- 叠加 HUD：左侧实时指标（速度、功率、踏频、心率、坡度），右下无底图路线总览（含当前位置）。
- 底部固定悬浮“坡度-距离”剖面图，持续展示当前位置指示。
- 新增沉浸模式专用“返回页面”按钮，解决顶部被隐藏后无返回入口的问题。
- 沉浸模式锁定页面滚动（`overflow: hidden`），去除上下左右滚动条。

## 2026-04-17 (蓝牙设备控制与坡度模拟重构)

### 蓝牙多服务融合连接
- 重构了 `power-meter.js`，从单一订阅 Cycling Power 服务改为并发探测与融合多服务 (`CP`, `CSC`, `FTMS`)。
- 解决了部分智能骑行台（如 T500）不在 Power 通道发送踏频数据的问题。现在系统能自动组合来自不同服务的数据，输出完整的 `{ power, cadence }`。

### FTMS 智能骑行台控制集成
- 新增 `trainer-ftms.js` 蓝牙适配器，完整实现了 FTMS (Fitness Machine Service) 控制协议，支持发送 `Set Target Inclination` (坡度模拟) 和 `Set Target Power` (ERG)。
- 在 `device-service.js` 中实现了“一键双连”逻辑：连接功率计后自动尝试获取骑行台的阻力控制权。

### 蓝牙指令防洪队列 (Throttle & Debounce)
- 针对智能骑行台无法处理高频下发指令（导致卡顿、不卸阻力）的问题，彻底重写了 FTMS `sendCommand`。
- 取消了强制等待 `0x80` 回复的阻塞逻辑，引入了带有 200ms 强制延时的异步发送队列。
- 增加了“丢弃过期指令”机制：当发送队列堵塞超过 2 条时，自动丢弃中间态，只下发最新坡度，确保骑行台阻力调整及时且不宕机。

### 坡度模拟算法与体验优化
- 修复了骑完路线终点后阻力卡在最大值的问题：超出路线距离后，目标坡度强制归零，提供放松体验。
- 在 `grade-sim-mode.js` 算法中，补全了地形平滑系数 (`smoothingFactor`) 和骑行台难度系数 (`difficultyPercent`) 的乘积计算，使阻力过渡更拟真。
- 增加了坡度变化阈值过滤：只有当目标坡度与当前坡度差异 $\ge 0.05\%$ 时才下发蓝牙指令，极大地降低了蓝牙信道压力。
