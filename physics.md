# Rider Tracker 物理模型说明与调参手册

本文基于当前实现文件 `src/domain/physics/cycling-model.js`，用于解释：

- 速度是怎么由功率/坡度/风速算出来的
- 为什么同功率在不同坡度下速度差异很大
- 后续如何调参数（`mass / cda / crr / windSpeed`）

---

## 1. 模型概览

当前模型分两条链路：

- `resolveSpeedTarget(...)`
  - 用二分法求“稳态速度”（某功率下，驱动力与阻力平衡时的速度）
  - 常用于“目标速度”估算
- `simulateStep(...)`
  - 每秒离散推进一次速度、距离、爬升、心率
  - 常用于整段路线模拟与实时骑行推进

两条链路现在使用同一套空气力方向逻辑，风速处理一致。

---

## 2. 核心公式

### 2.1 输入功率到有效功率

- `P_eff = P * (1 - drivetrain_loss)`
- 当前传动损失 `DRIVETRAIN_LOSS = 0.03`（3%）

### 2.2 坡度分解

- `slope = gradePercent / 100`
- `cosBeta = 1 / sqrt(1 + slope^2)`
- `sinBeta = slope * cosBeta`

### 2.3 阻力项（沿前进方向）

- 重力分量：`F_gravity = m * g * sinBeta`
- 滚阻：`F_roll = m * g * crr * cosBeta`
- 空气力（有方向）：`F_air = 0.5 * rho * cda * v_air * |v_air|`
  - `v_air = v + windSpeed`
  - 在当前符号约定下：
    - `windSpeed > 0`：逆风（相对风速变大，阻力增大）
    - `windSpeed < 0`：顺风（相对风速变小，甚至可产生助推）

### 2.4 稳态求解（`resolveSpeedTarget`）

- `P_resistive = max(0, (F_gravity + F_roll + F_air) * v)`
- 通过二分法在 `[0, 35 m/s]` 范围找 `P_resistive ~= P_eff`

### 2.5 动态推进（`simulateStep`）

- 驱动力近似：`F_drive = P_eff / v`（低速区做了封顶保护）
- 净力：`F_net = F_drive - (F_gravity + F_roll + F_air)`
- 加速度：`a = F_net / m`
- 速度更新：`v_next = clamp(v + a * dt, 0, 33.3)`

---

## 3. 参数与直觉

- `power`：最直接，增大后速度上升；但在高速区增速会边际递减（空气阻力二次增长）
- `gradePercent`：最敏感，尤其上坡会显著拉低速度
- `windSpeed`：非常敏感；顺/逆风 2~4 m/s 就会造成明显差异
- `cda`：高速段最敏感（平路、下坡更明显）
- `crr`：全速域都有影响，但通常幅度小于 `cda`
- `mass`：在平路影响相对小，上坡影响更大

## 7. 快速调参建议

### 7.1 如果你觉得“平路太快”

- 先增大 `cda`（例如 `0.32 -> 0.34/0.36`）
- 再检查 `windSpeed` 是否被设置成负值（顺风）

### 7.2 如果你觉得“爬坡太快”

- 增大 `mass`（骑手+车+装备总质量）
- 适当增大 `crr`（比如 `0.004 -> 0.0045`）

### 7.3 如果你觉得“下坡太快”

- 增大 `cda`（最有效）
- 若仍过快，可在业务层单独增加“下坡限速策略”而不改核心模型

## 8. 灵敏度参考（220W, 平路）

基准：`35.21 km/h`

- `mass`
  - `68kg -> 35.44`（`+0.23`）
  - `88kg -> 34.97`（`-0.23`）
- `cda`
  - `0.28 -> 36.72`（`+1.52`）
  - `0.36 -> 33.92`（`-1.29`）
- `crr`
  - `0.003 -> 35.66`（`+0.45`）
  - `0.005 -> 34.75`（`-0.45`）

结论：在平路稳态下，`cda` 比 `mass` 更敏感。

---

## 9. 当前模型边界与注意事项

- `resolveSpeedTarget` 搜索上限是 `35 m/s`（约 `126 km/h`）
- `simulateStep` 速度硬上限是 `33.3 m/s`（`120 km/h`）
- 低速起步区有保护逻辑（避免 `P/v` 在 `v≈0` 时发散）
- 心率模型是简化近似，适合趋势展示，不是医学模型

---

## 10. 推荐调参流程（实操）

1. 固定风速为 `0`，先把平路速度校准到你目标体感（主要调 `cda`）。
2. 再用 `+3% / +6%` 校准爬坡速度（主要调 `mass`，其次 `crr`）。
3. 最后引入 `windSpeed` 场景测试（`-2, 0, +2 m/s`）检查鲁棒性。
4. 若只是 UI 体验不希望出现极端高下坡速度，优先在展示层做限速提示，不直接改核心公式。
