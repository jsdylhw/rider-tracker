# Rider Tracker 项目阶段总结

## 1. 项目当前定位

当前项目已经从最初的蓝牙原型，演进为一个面向虚拟骑行场景的 Web 原型系统，核心链路已经基本打通：

- 支持手工路线和 GPX 导入
- 支持基于路线坡度的物理模拟
- 支持蓝牙心率带与功率计接入
- 支持实时骑行会话
- 支持真实地图底图与位置联动
- 支持导出 JSON 与 FIT

目前整体更适合定义为：

> 一个基于 Web Bluetooth、GPX 路线和基础物理模型的虚拟骑行原型平台

---

## 2. 已完成内容

### 2.1 路线与地图

已完成：

- 手工分段路线编辑
- GPX 文件导入
- GPX 轨迹点解析
- 累计距离、海拔、坡度生成
- 路线摘要统计
- 真实地图底图预览
- 起点、终点、当前位置 marker
- 已骑行轨迹高亮

说明：

- 当前路线数据已经不只是“显示用”，而是会直接参与速度计算、位置推进和 FIT 记录。

### 2.2 物理模拟

已完成：

- 恒定功率下的路线全程模拟
- 速度稳态求解
- 按秒推进距离、海拔、爬升
- 根据路线坡度动态变化速度
- 生成距离-时间曲线

当前逻辑：

- 输入一个功率
- 系统按整条路线模拟
- 自动得到总时长、平均速度、心率、距离等结果

### 2.3 蓝牙设备接入

已完成：

- 心率带连接与实时心率读取
- 功率计连接与实时功率读取
- 功率计踏频推导
- 实时均功率计算

说明：

- 蓝牙设备与模拟入口并存
- 当前已经具备“真实设备数据驱动虚拟骑行”的基础条件

### 2.4 实时骑行

已完成：

- 开始骑行 / 停止骑行流程
- 基于实时功率和路线坡度计算虚拟速度
- 基于实时心率更新会话数据
- 实时推进路线位置
- 自动进入独立骑行界面

当前骑行界面已包含：

- 骑行进度条
- 真实地图位置
- 当前路段
- 当前速度
- 当前坡度
- 平均心率
- 平均功率
- 最大功率
- TSS 基础估算

### 2.5 数据记录与导出

已完成：

- 模拟会话记录
- 实时骑行会话记录
- 本地保存最近一次会话
- JSON 导出
- FIT 导出

当前 FIT 已写入：

- 虚拟骑行活动类型
- 功率
- 心率
- 速度
- 距离
- 海拔
- 坡度
- GPS 经纬度

### 2.6 UI 与辅助能力

已完成：

- 主页面多区域布局
- 独立骑行界面
- 浮窗能力保留并复用
- 导出元信息表单
- 实时状态和结果面板

## 4. 各层职责

### app

负责应用编排：

- 初始化默认状态
- 接设备事件
- 接路线导入
- 接模拟与骑行流程
- 串联导出行为

### domain

负责核心业务逻辑：

- 路线构建
- GPX 解析
- 物理模型
- 模拟会话
- 实时骑行会话

### adapters

负责对外部接口和外围能力做适配：

- Web Bluetooth
- FIT 导出
- 本地存储

### ui

负责所有展示层：

- 主页面渲染
- 地图控制
- 骑行界面

### shared

负责通用工具：

- 时间格式化
- 数字格式化
- 下载工具

## 5. 当前已经形成的产品闭环

### 闭环 A：离线模拟

- 导入 GPX 或手工配置路线
- 输入恒定功率
- 运行整条路线模拟
- 查看结果
- 导出 JSON / FIT

### 闭环 B：实时骑行

- 导入 GPX
- 连接功率计
- 连接心率带
- 开始骑行
- 地图 marker 实时移动
- 查看实时指标
- 结束后保留会话结果

## 7. 当前阶段的主要限制

- 仍然是原型阶段，偏前端单机使用
- 地图底图方案已可用，但后续仍需增加更稳定的可切换方案
- FIT 导出还需要继续验证不同平台兼容性
- FTMS trainer 控制尚未接入
- 训练分析指标还不是完整训练平台级别

---

## 8. 推荐后续优先级

建议按下面顺序继续推进：

1. 优化真实地图体验与路线提示
2. 完善实时骑行指标体系
3. 接入 FTMS trainer
4. 做 Grade Simulation / ERG
5. 强化 FIT 兼容性与历史活动管理
6. 再做视频与风景联动

---

## 9. 一句话总结

当前项目已经不是单纯的蓝牙采集 demo，而是已经具备：

- 路线导入
- 地图展示
- 物理模拟
- 实时骑行
- 蓝牙接入
- FIT 导出

这六条主线的虚拟骑行原型系统。
